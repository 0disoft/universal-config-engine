# Universal Config Engine 1.0.0

Universal Config Engine `1.0.0` establishes the stable contract for local,
explainable configuration resolution across defaults, files, environment variables,
command-line arguments, coercion, and validation.

## Stable Contract

- Deterministic source priority and nested merge behavior.
- Value-level provenance for winning and overridden values.
- Bounded loaders, diagnostics, provenance, resolved paths, and report output.
- Secret-safe human and JSON diagnostics with report schema `0.2`.
- Stable `uce explain` and `uce validate` commands with exit codes `0` through `5`.
- Node.js `>=24`, ESM, and Windows/Linux compatibility gates.
- Optional Ajv and Zod validator adapters without validator dependencies in core.

## Packages

```sh
npm install @0disoft/universal-config-engine-core@1.0.0
npm install @0disoft/universal-config-engine-node@1.0.0
npm install --save-dev @0disoft/universal-config-engine-cli@1.0.0
```

Optional validator integrations are published as
`@0disoft/universal-config-engine-validator-ajv` and
`@0disoft/universal-config-engine-validator-zod`. Every package includes its own
installation and minimum-usage README.

## Breaking Change From 0.6

`ValidatorIssue` no longer exposes the ignored compatibility fields `category`,
`message`, `sourceId`, and `details`. Validators return `code`, `severity`, and an
optional normalized `path`. Core has ignored the removed fields since `0.3`, so the
change narrows the TypeScript declaration without changing runtime diagnostics.

See the [1.0 migration guide](https://github.com/0disoft/universal-config-engine/blob/main/docs/library/migration-1.0.md).

## Release Candidate Lineage

The stable package contents promote `1.0.0-rc.2` after its public review period.
RC2 added package-specific npm documentation, machine-enforced stable release
gates, reviewed release notes, and packed declaration migration checks without
changing the RC1 runtime or public API.

Diagnostic report schema `0.2`, CLI behavior, and the public declarations recorded
in the API snapshot are the compatibility baseline for the `1.x` line.
