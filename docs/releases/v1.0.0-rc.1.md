# Universal Config Engine 1.0.0-rc.1

This is the first release candidate for the stable `1.x` contract. It keeps the
engine local and stack-neutral: deterministic source precedence, value-level
provenance, bounded validation, and secret-safe diagnostics remain the product
boundary.

## Breaking Type Change

`ValidatorIssue` no longer exposes the ignored compatibility fields `category`,
`message`, `sourceId`, and `details`. Validators should return `code`, `severity`,
and an optional normalized `path`. Core has ignored the removed fields since `0.3`,
so runtime diagnostic behavior is unchanged.

See the [1.0 migration guide](https://github.com/0disoft/universal-config-engine/blob/main/docs/library/migration-1.0.md).

## Stable Candidate Contracts

- Diagnostic report schema remains `0.2` for `1.x`.
- CLI commands, flags, JSON envelopes, and exit codes remain stable for `1.x`.
- Node.js support remains `>=24` and ESM-only.
- Low-level helpers and programmatic CLI stages remain public.
- Validator typed output remains adapter-local and does not replace pipeline config.

Install this candidate by exact version or npm `next`; npm `latest` remains on the
previous stable release.
